## Night — Comprehensive Date/Interval Format Audit (2026-07-16 20:00–21:00 MYT)

### What Was Checked
Jack asked for a thorough check of all scripts and Google Tasks data to ensure consistent date format usage.

### Bugs Found & Fixed

**1. `compute_next_due()` timezone bug (task_commands.py)**
- **Problem**: `google_due_to_date` returned a UTC-aware datetime, but `datetime.replace(tzinfo=tz_utc)` stripped the timezone (made it naive). Naive UTC vs UTC-aware MYT comparison caused off-by-one date errors.
- **Fix**: Changed to `dt_obj.astimezone(tz_utc)` — preserves timezone correctly.
- **Impact**: `half-yearly` from Apr 10 would return Oct 9 instead of Oct 10; monthly/bi-monthly etc. could also misfire near midnight MYT.

**2. `half-yearly` used 183 hardcoded days**
- **Problem**: 183 days ≠ exactly 6 months (e.g., Apr 10 + 183 days = Oct 9, but 6 months = Oct 10).
- **Fix**: Changed to `relativedelta(months=6)`.

**3. `bi-weekly` missing from `day_based` table**
- Was in `RRULE_FREQ`/`RRULE_INT` but had no actual computation path.
- **Fix**: Added `6` to `day_based`.

**4. `daily` missing from `day_based` table**
- **Fix**: Added `1` to `day_based`.

**5. `quarterly` missing from `RRULE_INT`**
- **Fix**: Added `'quarterly': 3`.

**6. `REC_TAG_RE` generic fallback could match `1-yearly`**
- `\d+-years?` could match `yearly` → would never reach the explicit tag.
- **Fix**: Changed to `\d+-years?ly?` — requires a number before `year`.

## Critical Lesson: Never Kill the Lobster Server Process

The lobster server (`tsx src/server.ts`) running on port 3099 shares the same process tree as OpenClaw. When I kill the lobster server process directly (via `kill`, `pkill`, `fuser` on port 3099, etc.), I also kill OpenClaw itself — it goes dark and requires manual restart.

**What happened:** Restarting the lobster server for the `/validate` endpoint update crashed OpenClaw twice.

**Rule:** I must NEVER use `kill`, `pkill`, `fuser`, or any other direct process termination on any process using port 3099, or named `tsx src/server.ts`, or that is the lobster server.

**When the lobster server needs restarting (for code changes):**
- Ask Jack to run the restart command from a terminal: `cd /home/ubuntu/works/lobster-automation/server-deployment && pkill -f "tsx src/server.ts" && sleep 2 && node_modules/.bin/tsx src/server.ts &`
- The lobster server auto-reloads via `tsx watch` when .ts files change — only a full process restart needs the above
- If `/validate` or other new endpoints don't appear to work, first check if `tsx watch` already picked up the change (it usually does)

**Confirm before killing any process if unsure:**
```bash
ps aux | grep -E "tsx|node.*server" | grep -v grep
```
If the process list includes anything that might be shared with OpenClaw, do not kill it — ask Jack instead.
