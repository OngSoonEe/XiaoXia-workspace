## lobster-automation relay project (PHP relay on ewizt.com)

Goal: replace ngrok with a self-hosted relay queue on ewizt.com, so WSL doesn't need ngrok.

### Architecture
- External callers POST to ewizt.com webhook endpoint
- ewizt.com queues webhook payload in SQLite
- WSL lobster server long-polls ewizt.com for queued webhooks (near-instant, no polling delay)
- WSL processes and posts result back; caller checks result via GET

### PHP files written (in ~/works/lobster-automation/relay-php/)
- `shared.php` — SQLite queue helper (DB at `queue.sqlite`, TOKEN=`NsufkdwqJWfr4GwFko7R`)
- `webhook.php` — POST `/relay/webhook/:name` — receives+queues webhooks, returns run_id
- `poll.php` — GET `/relay/poll` — long-polls (up to 55s) for queued webhooks
- `result.php` — GET/POST `/relay/result/:id` — WSL posts result, caller checks result
