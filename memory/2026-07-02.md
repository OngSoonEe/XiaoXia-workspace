## Cloudflare Tunnel — DNS Resolution Issue

### The Problem
- Tunnel process is healthy (4 edge connections, `active (running)`)
- CNAME at mschosting: `automation.ewizt.com → 86e7097f-...cfargotunnel.com` ✅
- But `cfargotunnel.com` subdomain has **no public A record** — only private AAAA (`fd10:aec2:5dae::`)
- Root cause: Cloudflare zone for `ewizt.com` is **`pending`** — nameservers still point to `ns137/138.mschosting.cloud`
- Phone browser gets Error 1001 (DNS resolution error at CF edge)

### Attempted Fixes
1. Added A record `104.16.0.1` at Cloudflare (proxied) → still Error 1001 because zone is pending
2. Changed mschosting from CNAME to A `104.16.0.1` → same issue
3. Quick Tunnel (`trycloudflare.com`) — worked but URL changes on restart
4. Recreated tunnel with new credentials — tunnel healthy but DNS still broken

### Key Learning

- **Cannot test tunnel from AI browser** — sandbox blocks private IP navigation
- **Only reliable test** is phone on LTE or online DNS checker
- **The fix requires switching nameservers** at registrar to Cloudflare's NS, but Jack can't do this because other apps depend on mschosting DNS

### Current State
- Tunnel: healthy, 4 connections, running as systemd service
- DNS: `automation.ewizt.com` → A `104.16.0.1` at mschosting → CF edge → Error 1001
- Server: running on port 3099, serving locally
- **Blocked** on nameserver switch or alternative routing solution

## Cloudflare Tunnel — DNS Resolution Issue (Updated)

### The Problem
- Tunnel process is healthy (4 edge connections, `active (running)`)
- Cloudflare zone for `ewizt.com` is **`pending`** — NS delegation in progress at registrar
- Jack added NS records at mschosting for `automation.ewizt.com → dante.ns.cloudflare.com + gail.ns.cloudflare.com`
- But zone is `full` type (needs full nameserver switch) not `partial` (CNAME-only)
- All tunnel services reject custom Host headers — security feature prevents `automation.ewizt.com` from routing through `*.trycloudflare.com` or `*.loca.lt`

### Attempted Fixes (July 2, 2026)
1. **A record `104.16.0.1`** at mschosting → hangs (CF edge can't find zone config, pending)
2. **localtunnel** — works via direct URL, but CNAME approach fails because localtunnel checks Host header and only accepts its own subdomain
3. **cloudflared tunnel** — same Host header rejection issue
4. **NS delegation** — set at mschosting, but zone is `full` type and still `pending`
5. **CNAME at mschosting → `*.loca.lt`** — 503 Tunnel Unavailable because tunnel service rejects `automation.ewizt.com` Host header

### Current Working Solution
- **Direct cloudflared URL** works: `https://covering-garcia-blowing-privacy.trycloudflare.com/health` → `{"ok":true}`
- Service: `lobster-localtunnel.service` (actually running cloudflared now)
- URL changes on restart — no way around this with free tunnel services
- CNAME at mschosting currently pointing to: `lobster-d68639f4.loca.lt` (not working due to Host header)

### Fundamental Blocker
All tunnel services (localtunnel, cloudflared, serveo, localhost.run) validate the Host header and only accept their own domain. CNAME routing doesn't work because:
1. Browser → `automation.ewizt.com` → CNAME → `tunnel-service.com`
2. Browser sends `Host: automation.ewizt.com` to `tunnel-service.com`
3. Tunnel service rejects because it expects `Host: <assigned-subdomain>.tunnel-service.com`

### Real Fix Requires
- **Option A:** Wait for Cloudflare zone `pending` → `active` (zone auto-activates when NS delegation propagates)
- **Option B:** Switch full nameservers from mschosting to Cloudflare (zone becomes `full` and active)
- **Option C:** Use Cloudflare Workers as a proxy layer (rewrites Host header) — but zone needs to be active

### Key Learning
- **Cannot test tunnel from WSL** — WSL DNS resolves `automation.ewizt.com` to private IPv6
- **Cannot test tunnel from AI browser** — sandbox blocks private IP navigation
- **Only reliable test** is phone on LTE or online DNS checker
- **localtunnel `--subdomain` flag** doesn't guarantee the subdomain — service may give different one on restart
