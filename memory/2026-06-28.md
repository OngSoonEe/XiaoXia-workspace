# 2026-06-28 Daily Notes

## Daily News Site — Big Fix Day

### What we fixed today

**1. Timezone bug in `normalize-news.ts`**
- Root cause: `toLocaleString('en-US', {timeZone: 'Asia/Kuala_Lumpur'})` + `new Date(klNow)` was interpreting the date string as local (UTC) time, giving wrong `todayStart`
- Fix: Direct UTC+8 arithmetic — get current UTC, add 8h, extract UTC year/month/day for KL date, construct `todayStart = new Date('YYYY-MM-DDT00:00:00Z')`
- Effect: `yesterdayTitles` went from 0 to 377 entries; cross-day dedup finally worked

**2. Age filter for RSS articles in `select-news.ts`** *(the big one)*
- Root cause: RSS articles (from `rss-supplement.json`) bypassed `normalize-news.ts` entirely — no age filter, no cross-day dedup
- The model was selecting old RFA articles (China robots Jun 17, HK bookstore Jun 25, Venezuela earthquake Jun 27) that had never been filtered
- Fix: Added age filter in pool-building loop of `select-news.ts` — filter out articles >48h old (for global/asia) or >72h (malaysia) based on `publishedAt`
- Age filter now applies to BOTH NewsAPI and RSS articles in the selection pool

**3. FTP greeting consumer in `upload-ftp.ts`**
- Server was sending "220" greeting which was consumed as the USER command response
- Fix: Added `socket.once('data', ...)` at start of `connect` callback to consume and discard the greeting before sending USER

**4. Made `main()` async in `upload-ftp.ts`**
- Needed to support `await` inside Promise executor
